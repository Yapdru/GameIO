DONECLAUDE HANDOFF — GAMEIO NEXT STEPS

Status:
Claude has finished its current pass. This file is the handoff for the next clean GameIO build.

Main rule:
GameIO is a multiplayer arcade universe, not a fake realistic driving demo.

Core loop:
1. Player opens GameIO.
2. Player creates or joins a room.
3. Player chooses a clean avatar.
4. Player enters multiplayer lobby.
5. Host selects a playlist of games.
6. Players play games one by one.
7. Scores update after each game.
8. Final winner is shown.
9. Players can replay or return to lobby.

Multiplayer requirement:
Multiplayer must stay.
Use Firebase Realtime Database if config exists.
Quick Play/offline mode may exist, but it must not replace multiplayer.

Required games:
- Fishana Evolution
- Cars / Drift
- Badaam Saat
- Space Dash
- Obby Run
- Quiz / Riddle
- Math Dash

Clean architecture:
Do not keep everything in one giant file.
Use separate modules.

Recommended files:
index.html
style.css
app.js
firebase.js
game-data.js
avatars.js
lobby.js
ui.js

games/fishana.js
games/cars.js
games/badaam.js
games/space.js
games/obby.js
games/quiz.js
games/mathdash.js

Phase 1 — Clean shell:
- Start screen
- Create Room
- Join Room
- Quick Play
- Avatar screen
- Lobby screen
- Game selector
- Results screen

Phase 2 — Multiplayer basics:
- Create room code
- Join room code
- Sync player names
- Sync avatars
- Sync scores
- Host starts game
- Host starts next round

Phase 3 — Playable games:
Every game must be playable, even if simple.
Each game must return a score.
Each game must end cleanly and return to results/lobby.

Fishana Evolution:
Move fish, collect pearls, avoid enemy, evolve, score.

Cars / Drift:
Arcade driving, steering, boost, drift score, checkpoints or laps.

Badaam Saat:
Card table, valid move highlight, play/pass, scoring.

Space Dash:
Avoid obstacles, collect stars, boost, score.

Obby Run:
Jump/dodge, platforms, checkpoints, score.

Quiz / Riddle:
Multiple choice, timer, correct score.

Math Dash:
Random math, timer, increasing difficulty, score.

Phase 4 — 3D after gameplay:
3D is allowed, but it must support the game loop.
Do not make the entire game a driving-only demo.

Final 3D target:
- Clean stylized 3D lobby
- Game portals/floors for Fishana, Cars, Badaam Saat, Space, Obby, Quiz, Math Dash
- Simple avatar movement
- Smooth camera follow
- Touch joystick support
- Light blue/yellow arcade look
- Multiplayer players visible in lobby
- Portal entry starts the chosen game

3D ideas that may be kept as inspiration:
- Ver XP / Ver 11 Pro cinematic intro
- First-person arrival feeling
- Drive toward an elevator
- Glass elevator as transition to lobby
- Music by scene
- GAMEIO / VER XP style logo scramble
- Simulated in India identity
- Forza-inspired feeling of camera weight, lighting, and reveal moments

How to use the Forza inspiration safely:
Do not copy Forza assets, names, cars, music, maps, UI, or brand style.
Use only the design lessons:
- slow reveal
- strong lighting
- camera weight
- meaningful movement
- environment detail
- satisfying sound
- clear objective
- polished transitions

Music lesson:
Scene-based music was attempted:
- driving music
- elevator music
- lobby music
This is a good idea, but avoid copyrighted music in final production unless the owner supplies licensed assets. Use simple generated sounds or safe audio files.

What went wrong in the 3D experiment:
- The build tried to become Forza too quickly.
- 3D became the main feature before gameplay was stable.
- The original games got hidden behind the driving/elevator demo.
- Files became too large and hard to maintain.
- Visuals looked like placeholder blocks because detail was added too late.
- The root page had to be forced into 3D, which damaged the original app flow.

What must not happen again:
- Do not create one huge v8-3d.js-style file.
- Do not replace the whole GameIO with a 3D cinematic.
- Do not remove game selection.
- Do not remove multiplayer.
- Do not hide Fishana, Cars, Badaam Saat, and other games behind a demo.
- Do not overbuild fake realism.

Correct 3D build order:
1. Build multiplayer lobby in 2D first.
2. Make Fishana playable.
3. Make Cars playable.
4. Add Badaam Saat.
5. Add results and next-game flow.
6. Add a stylized 3D lobby layer.
7. Add portals that launch real games.
8. Add cinematic transitions.
9. Add sound and particles.
10. Add first-person arrival only as an intro option, not the whole game.

Visual style:
Light blue, yellow, white, navy accents.
Clean, bright, playful, smooth.
No trash emoji gameplay.
No ugly placeholder cubes.
No fake AAA overbuild.

Acceptance criteria:
1. Create room works.
2. Join room works.
3. Players appear in lobby.
4. Host starts Fishana.
5. Fishana can be played and scored.
6. Scores sync.
7. Results show winner.
8. Next game can start.
9. Cars and Badaam Saat are reachable from the same lobby.
10. 3D lobby, if added, supports the game loop instead of replacing it.
11. GameIO still feels like many games in one multiplayer arcade universe.

Final principle:
Gameplay first. Multiplayer first. Clean structure first. 3D and polish later.
